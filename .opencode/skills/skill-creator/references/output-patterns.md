# Output Patterns

Use these patterns when repo-local skills need to produce consistent, high-quality output.

## Template Pattern

Provide templates for output format. Match the level of strictness to the task.

**For stricter skill outputs:**

```markdown
## Output

- current behavior
- expected behavior
- concrete risks or mismatches
- smallest safe next step
```

**For more flexible skill outputs:**

```markdown
## Output

Use this default structure, but adapt to the workflow:
- summary
- important findings
- next action
```

## Examples Pattern

When a skill needs a very specific answer style, add one or two short examples instead of long prose.

## Repo Audit Pattern

For engineering audit skills, prefer a short output contract like:

```markdown
## Output

- current behavior
- expected behavior
- concrete risks or mismatches
- smallest safe next step
```
