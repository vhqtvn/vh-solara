## Scope

Headline: the warm-open tunnel-perf investigation — a two-slice fix plus the
honest-metric relabel and per-turn stats.

### 1. Warm-open tunnel-perf (two-slice fix)

- **Indicator (Part A):** a `.dot.refreshing` signals when a warm session paints cached data while its fresh snapshot is in flight (no more silent ~5s stale→fresh swap).
- **Root cause (slice a):** the ~5s was **tunnel transport**, not server compute. Capped `refreshOpenSessions()` fan-out from an unbounded `Promise.all` to bounded concurrency (`REFRESH_CONCURRENCY = 3`, tunable) to protect the single yamux-over-WebSocket tunnel from head-of-line / bandwidth contention.
- **Metric honesty:** relabeled the FE `server` latency metric to `snap` — the old name measured `onopen → first snapshot event`, which structurally included tunnel transit; `snap` is the honest end-to-end label.

The full diagnosis, shipped-commit table, diagnostic checklist, and deferred
follow-ups (slice (b) payload shrink, option D trigger debounce, formally
retiring the store hypothesis under token-storm load) are in
[`2026-07-08-warm-open-tunnel-perf-investigation.md`](2026-07-08-warm-open-tunnel-perf-investigation.md) —
referenced here, not restated.

### 2. Per-turn tok/s + TTFT stats

- `tok/s` and TTFT shown per assistant turn behind a hover info icon, with the hover affordance styled and TTFT clamped to non-negative (consistency with the tok/s guard).

### 3. Diagnostic ergonomics + UX hardening

- `GET /vh/snapshot` `dur_ms` logging (VH_DEBUG-gated) for snapshot-latency diagnosis.
- `make build-debug` target with forced debug logging via ldflags.
- Chat stays pinned when the composer textarea auto-grows; OpenCode update UX stabilized with a unified restart flow.
